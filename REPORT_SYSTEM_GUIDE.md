## 📋 功能概述

报表生成系统是EOMS的数据分析扩展功能，提供自动化的监控数据统计和报表生成能力，帮助用户快速了解系统运行状况。

### 核心功能

1. **预定义报表** - 日报、周报、月报自动生成
2. **自定义报表** - 灵活选择时间范围和节点
3. **数据统计** - 自动计算平均值、最大值、最小值
4. **报表导出** - JSON格式导出
5. **报表管理** - 查看、删除历史报表

---

## 📊 报表类型

### 1. 日报 (Daily Report)
- **时间范围**: 昨天（00:00 - 23:59）
- **适用场景**: 每日例行检查
- **数据粒度**: 1小时聚合

### 2. 周报 (Weekly Report)
- **时间范围**: 过去7天
- **适用场景**: 周度趋势分析
- **数据粒度**: 1小时聚合

### 3. 月报 (Monthly Report)
- **时间范围**: 过去30天
- **适用场景**: 月度总结汇报
- **数据粒度**: 1小时聚合

### 4. 自定义报表 (Custom Report)
- **时间范围**: 用户自定义
- **适用场景**: 特定时间段分析
- **数据粒度**: 1小时聚合

## 🚀 快速开始

### 1. 访问报表管理

登录后，在侧边栏点击"报表管理"菜单项（📊图标）。

```
侧边栏 → 报表管理
```

### 2. 生成报表

#### 方式一：生成预定义报表

1. 点击"生成报表"按钮
2. 选择报表类型（日报/周报/月报）
3. 选择要统计的节点（可多选）
4. 点击"生成报表"

#### 方式二：生成自定义报表

1. 点击"生成报表"按钮
2. 选择报表类型为"自定义"
3. 选择要统计的节点
4. 选择开始时间和结束时间
5. 点击"生成报表"

### 3. 查看报表

1. 在报表列表中找到目标报表
2. 点击"查看"按钮（👁️图标）
3. 查看报表详情和统计数据

### 4. 导出报表

1. 在报表列表或详情页面
2. 点击"下载"按钮（⬇️图标）
3. 报表将以JSON格式下载到本地

### 5. 删除报表

1. 在报表列表中找到要删除的报表
2. 点击"删除"按钮（🗑️图标）
3. 确认删除操作

**注意**: 只有管理员或报表创建者可以删除报表

## 🔧 后端API接口

### 报表管理接口

#### 获取所有报表
```http
GET /api/reports?limit=50
Authorization: Bearer {token}
```

#### 获取指定报表
```http
GET /api/reports/{reportId}
Authorization: Bearer {token}
```

#### 生成自定义报表
```http
POST /api/reports/generate
Authorization: Bearer {token}
Content-Type: application/json

{
  "nodeIds": ["node1", "node2"],
  "startTime": "2026-02-21T00:00:00Z",
  "endTime": "2026-02-22T00:00:00Z",
  "metrics": ["cpu", "memory", "disk", "network"]
}
```

#### 生成预定义报表
```http
POST /api/reports/generate/{type}
Authorization: Bearer {token}
Content-Type: application/json

{
  "nodeIds": ["node1", "node2"]
}
```

**type**: daily | weekly | monthly

#### 删除报表
```http
DELETE /api/reports/{reportId}
Authorization: Bearer {token}
```

#### 获取报表摘要
```http
GET /api/reports/{reportId}/summary
Authorization: Bearer {token}
```

#### 获取报表类型列表
```http
GET /api/reports/meta/types
Authorization: Bearer {token}
```

## 📊 数据存储

报表数据存储在Redis中，使用以下数据结构：

### 报表元数据
```
Key: report:{reportId}
Type: Hash
Fields:
  - id: 报表ID
  - type: 报表类型
  - nodeIds: 节点ID列表（JSON）
  - startTime: 开始时间
  - endTime: 结束时间
  - metrics: 指标列表（JSON）
  - statistics: 统计信息（JSON）
  - createdAt: 创建时间
  - createdBy: 创建者ID
TTL: 30天
```

### 报表列表
```
Key: reports
Type: Set
Members: 所有报表ID
```

## 💡 使用建议

### 1. 定期生成报表

建议每天生成日报，每周生成周报，每月生成月报，以便：
- 及时发现系统异常
- 跟踪性能趋势
- 积累历史数据

### 2. 合理选择节点

- 重要节点：每天生成报表
- 一般节点：每周生成报表
- 测试节点：按需生成报表

### 3. 报表保留策略

- 报表自动保留30天
- 重要报表建议及时导出保存
- 定期清理不需要的报表

### 4. 数据分析建议

- 关注平均值趋势
- 注意最大值异常
- 对比不同时间段数据
- 结合告警记录分析

## ⚠️ 注意事项

### 1. 数据可用性

- 报表依赖InfluxDB历史数据
- 如果历史数据不足，统计可能不准确
- 建议至少运行24小时后再生成报表

### 2. 性能影响

- 生成报表需要查询大量历史数据
- 避免同时生成多个大型报表
- 建议在系统空闲时生成报表

### 3. 存储空间

- 报表数据占用Redis存储空间
- 自动过期时间为30天
- 可根据需要调整过期时间

### 4. 权限控制

- 所有用户都可以生成和查看报表
- 只有管理员或创建者可以删除报表
- 报表数据包含敏感信息，注意保密

## 🐛 常见问题

### Q1: 报表生成失败怎么办？

**A**: 可能的原因：
1. 选择的节点没有历史数据
2. InfluxDB连接失败
3. 时间范围选择不当

**解决方法**：
- 检查节点是否在线并有数据上报
- 检查InfluxDB服务状态
- 调整时间范围重新生成

### Q2: 报表数据为什么不准确？

**A**: 可能的原因：
1. 数据采集不完整
2. Agent断线导致数据缺失
3. 时间范围内数据量不足

**解决方法**：
- 确保Agent持续运行
- 检查数据采集日志
- 选择数据完整的时间段

### Q3: 如何导出PDF格式报表？

**A**: 当前版本仅支持JSON格式导出。PDF导出功能计划在后续版本中实现。

### Q4: 报表可以定时自动生成吗？

**A**: 当前版本需要手动生成。自动定时生成功能计划在后续版本中实现。

### Q5: 报表数据可以发送邮件吗？

**A**: 当前版本不支持邮件发送。邮件推送功能计划在后续版本中实现。
