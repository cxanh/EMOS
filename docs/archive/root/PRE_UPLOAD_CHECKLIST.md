# 上传前检查清单

在将项目上传到GitHub之前，请仔细检查以下事项：

---

## 🔒 安全检查

### 1. 环境变量文件

- [ ] `backend/.env` 已添加到 `.gitignore`
- [ ] `frontend/.env` 已添加到 `.gitignore`
- [ ] 已创建 `backend/.env.example` 示例文件
- [ ] 已创建 `frontend/.env.example` 示例文件

### 2. 敏感信息检查

检查以下文件中是否包含敏感信息：

- [ ] 数据库密码
- [ ] API密钥
- [ ] JWT密钥
- [ ] 邮箱密码
- [ ] 钉钉Webhook地址
- [ ] 真实的IP地址
- [ ] 个人身份信息

**需要检查的文件**:
```
backend/.env
backend/config/*.js
backend/services/*.js
frontend/.env
agent/config.yaml
```

### 3. 个人信息检查

- [ ] 毕业设计文档已排除（`graduation_design/`）
- [ ] 个人简历、学号等信息已移除
- [ ] 真实姓名、联系方式已移除或替换

## 🆘 如果发现问题

### 已经上传了敏感信息

1. 立即删除包含敏感信息的提交
2. 修改所有泄露的密码和密钥
3. 参考 `GITHUB_UPLOAD_GUIDE.md` 中的解决方案

### 文件太大

1. 使用 `.gitignore` 排除大文件
2. 考虑使用 Git LFS
3. 将大文件存储在其他地方

### 提交了错误的文件

1. 使用 `git rm --cached` 移除文件
2. 添加到 `.gitignore`
3. 重新提交
