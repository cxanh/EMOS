## 📋 测试步骤

### 测试1: 创建告警规则 ✅

**步骤**:
1. 打开浏览器访问 `http://localhost:5173`
2. 登录系统（admin/admin）
3. 点击侧边栏 "🔔 告警管理"
4. 点击 "➕ 新建规则" 按钮
5. 填写表单：
   ```
   规则名称: CPU使用率过高测试
   监控节点: 所有节点
   监控指标: CPU使用率
   条件: 大于 (>)
   阈值: 80
   持续时间: 60
   通知方式: ✓ WebSocket推送
   ```
6. 点击 "创建" 按钮

**预期结果**:
- ✅ 弹出提示框："✅ 规则创建成功！"
- ✅ 对话框自动关闭
- ✅ 规则列表中立即显示新规则
- ✅ 规则状态为启用（绿色圆点）

**如果失败**:
- 检查浏览器控制台（F12）是否有错误
- 检查后端日志 `backend/logs/combined.log`
- 运行测试脚本：`node backend/test-alert-create.js`

---

### 测试2: 编辑告警规则 ✅

**步骤**:
1. 在规则列表中找到刚创建的规则
2. 点击规则右侧的 "✏️" 编辑按钮
3. 修改阈值为 `90`
4. 点击 "保存" 按钮

**预期结果**:
- ✅ 弹出提示框："✅ 规则更新成功！"
- ✅ 对话框自动关闭
- ✅ 规则列表中阈值更新为 90%

---

### 测试3: 启用/禁用规则 ✅

**步骤**:
1. 找到任意一个规则
2. 点击规则左侧的 "⏸" 或 "▶" 按钮

**预期结果**:
- ✅ 弹出提示框："✅ 规则已禁用！" 或 "✅ 规则已启用！"
- ✅ 规则状态图标立即变化（● 变为 ○，或相反）
- ✅ 规则卡片样式变化（禁用时变灰）

---

### 测试4: 删除告警规则 ✅

**步骤**:
1. 找到测试规则
2. 点击规则右侧的 "🗑️" 删除按钮
3. 在确认对话框中点击 "确定"

**预期结果**:
- ✅ 弹出提示框："✅ 规则删除成功！"
- ✅ 规则立即从列表中消失

## 🔧 故障排除

### 问题1: 创建规则后没有提示

**检查**:
```javascript
// 打开浏览器控制台（F12）
// 查看是否有JavaScript错误
```

**解决**:
- 清除浏览器缓存
- 重新加载页面（Ctrl+F5）
- 检查前端代码是否最新

### 问题2: 规则列表不更新

**检查**:
```bash
# 检查后端日志
tail -f backend/logs/combined.log

# 检查Redis连接
redis-cli ping
```

**解决**:
- 重启后端服务
- 检查Redis是否运行
- 运行测试脚本验证API

### 问题3: 提示显示但规则不在列表中

**检查**:
```bash
# 使用测试脚本验证
node backend/test-alert-create.js

# 检查Redis数据
redis-cli
> KEYS alert:rule:*
> HGETALL alert:rule:<rule_id>
```

**解决**:
- 检查后端alertService.js
- 检查Redis数据存储
- 查看后端错误日志
