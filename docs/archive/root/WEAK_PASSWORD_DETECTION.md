# 弱口令检测功能实现

## 功能概述

为登录页面添加了实时的弱口令检测和密码强度评估功能，帮助用户识别和避免使用弱密码。

## 功能特性

### 1. 实时密码强度评估
- 输入密码时实时显示密码强度
- 强度等级：弱密码、中等强度、强密码
- 可视化强度条（红色/橙色/绿色）

### 2. 弱口令检测
检测以下类型的弱口令：

#### 常见弱口令列表
- `123456`, `123456789`, `password`, `admin`, `root` 等
- 包含30+个常见弱密码

#### 模式检测
- 纯数字且长度小于8位
- 连续数字（如 `1234`, `5678`）
- 重复字符（如 `111111`, `aaaaaa`）

### 3. 视觉警告
- 检测到弱口令时显示黄色警告框
- 带有警告图标和提示文字
- 震动动画效果引起注意

## 密码强度评分规则

### 评分标准（总分100分）

1. **长度评分**（最高50分）
   - 8位及以上：+25分
   - 12位及以上：+15分
   - 16位及以上：+10分

2. **字符类型**（最高50分）
   - 包含小写字母：+15分
   - 包含大写字母：+15分
   - 包含数字：+15分
   - 包含特殊字符：+20分

3. **多样性加分**（最高20分）
   - 包含3种字符类型：+10分
   - 包含4种字符类型：+10分

4. **弱口令惩罚**
   - 如果匹配弱口令规则，强制降低到30分以下

### 强度等级划分

- **弱密码**（0-39分）：红色
- **中等强度**（40-69分）：橙色
- **强密码**（70-100分）：绿色

## 安全建议

### 推荐的密码策略

1. **最低要求**
   - 至少8个字符
   - 包含大小写字母
   - 包含数字
   - 包含特殊字符

2. **避免使用**
   - 常见单词或短语
   - 个人信息（生日、姓名等）
   - 键盘序列（qwerty等）
   - 重复或连续字符

3. **最佳实践**
   - 使用12位以上的密码
   - 混合使用4种字符类型
   - 定期更换密码
   - 不同系统使用不同密码

### 后端验证

建议在后端也添加密码强度验证：

```javascript
// backend/routes/auth.js
function validatePasswordStrength(password) {
  // 检查长度
  if (password.length < 8) {
    return { valid: false, message: '密码长度至少8位' };
  }
  
  // 检查弱口令
  const weakPasswords = [...];
  if (weakPasswords.includes(password.toLowerCase())) {
    return { valid: false, message: '密码过于简单，请使用更强的密码' };
  }
  
  // 检查字符类型
  const hasLower = /[a-z]/.test(password);
  const hasUpper = /[A-Z]/.test(password);
  const hasNumber = /\d/.test(password);
  const hasSpecial = /[!@#$%^&*]/.test(password);
  
  const types = [hasLower, hasUpper, hasNumber, hasSpecial].filter(Boolean).length;
  
  if (types < 3) {
    return { valid: false, message: '密码必须包含至少3种字符类型' };
  }
  
  return { valid: true };
}
```
