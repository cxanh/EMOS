## 功能概述

自定义仪表盘是EOMS系统的高级可视化功能，允许用户根据自己的需求自由组合和布局各种监控组件，创建个性化的监控视图。

### 核心特性

- 🎨 **灵活布局**: 自由添加、删除和排列组件
- 📊 **多种图表**: 6种专业可视化组件
- 💾 **自动保存**: 布局自动保存到本地
- 📱 **响应式**: 自适应不同屏幕尺寸
- ⚡ **实时更新**: 数据实时刷新

## 快速开始

### 1. 访问仪表盘

1. 登录EOMS系统
2. 在左侧导航栏找到"自定义仪表盘"（🎨图标）
3. 点击进入仪表盘页面

### 2. 添加第一个组件

1. 点击页面右上角的"添加组件"按钮
2. 在弹出的对话框中：
   - 选择组件类型（如"CPU仪表盘"）
   - 输入组件标题（如"主服务器CPU"）
   - 选择要监控的节点
   - 选择组件大小（小/中/大）
3. 点击"添加"按钮
4. 组件将立即显示在仪表盘上

### 3. 查看实时数据

- 组件添加后会自动开始显示实时数据
- 数据每秒自动更新
- 鼠标悬停在图表上可查看详细信息

## 组件类型

### 1. CPU仪表盘 💻

**适用场景**: 实时监控CPU使用率

**特点**:
- 半圆仪表盘设计
- 颜色分级（绿色正常、黄色警告、红色危险）
- 显示当前值和平均值
- 平滑动画效果

**推荐大小**: 小 (1x1) 或 中 (2x1)

**使用建议**:
- 用于关键服务器的CPU监控
- 配合告警规则使用
- 多个节点对比时使用小尺寸

### 2. 内存仪表盘 🧠

**适用场景**: 实时监控内存使用率

**特点**:
- 半圆仪表盘设计
- 蓝色系配色
- 显示当前值和平均值
- 实时动画更新

**推荐大小**: 小 (1x1) 或 中 (2x1)

**使用建议**:
- 监控内存密集型应用
- 及时发现内存泄漏
- 容量规划参考

### 3. 磁盘饼图 💾

**适用场景**: 磁盘空间使用分析

**特点**:
- 环形饼图展示
- 已使用/可用空间对比
- 渐变色彩效果
- 百分比显示

**推荐大小**: 小 (1x1) 或 中 (2x1)

**使用建议**:
- 监控磁盘空间使用
- 预防磁盘满
- 存储容量规划

### 4. 网络趋势图 🌐

**适用场景**: 网络流量趋势分析

**特点**:
- 双线趋势图
- 接收/发送流量对比
- 面积填充效果
- 时间轴展示

**推荐大小**: 中 (2x1) 或 大 (2x2)

**使用建议**:
- 监控网络流量变化
- 发现异常流量
- 带宽使用分析

### 5. 系统雷达图 🎯

**适用场景**: 系统综合健康状态

**特点**:
- 四维雷达图
- CPU/内存/磁盘/网络综合展示
- 渐变填充效果
- 一目了然的健康状态

**推荐大小**: 中 (2x1) 或 大 (2x2)

**使用建议**:
- 系统整体健康检查
- 快速发现瓶颈
- 性能平衡分析

### 6. 统计卡片 📋

**适用场景**: 多指标快速概览

**特点**:
- 四宫格卡片布局
- 实时指标显示
- 趋势指示器（↑↓→）
- 渐变背景效果

**推荐大小**: 中 (2x1) 或 大 (2x2)

**使用建议**:
- 首页概览
- 多指标对比
- 快速状态检查

## 使用教程

### 基础操作

#### 添加组件

1. **打开添加对话框**
   - 点击"添加组件"按钮
   - 或在空仪表盘点击"添加第一个组件"

2. **配置组件**
   ```
   组件类型: 选择要添加的组件类型
   组件标题: 输入有意义的标题（如"生产服务器CPU"）
   选择节点: 选择要监控的节点（可选"所有节点"）
   组件大小: 根据需要选择大小
   ```

3. **确认添加**
   - 点击"添加"按钮
   - 组件立即显示在仪表盘上

#### 删除组件

1. **进入编辑模式**
   - 点击"编辑布局"按钮
   - 组件边框变为虚线

2. **删除组件**
   - 点击组件右上角的❌按钮
   - 组件立即被删除

3. **退出编辑模式**
   - 点击"完成编辑"按钮
   - 布局自动保存

#### 重置布局

1. **点击重置按钮**
   - 点击"重置布局"按钮

2. **确认操作**
   - 在确认对话框中点击"确定"
   - 所有组件将被清空

3. **重新添加**
   - 从零开始添加组件

### 高级技巧

#### 1. 创建监控大屏

**目标**: 创建一个全面的监控大屏

**步骤**:
1. 添加多个统计卡片组件（小尺寸）
2. 添加关键指标的仪表盘（中尺寸）
3. 添加趋势图（大尺寸）
4. 添加雷达图（中尺寸）

**布局建议**:
```
┌──────────┬──────────┬──────────┐
│ 统计卡片 │ 统计卡片 │ 统计卡片 │
├──────────┴──────────┴──────────┤
│        网络趋势图（大）         │
├──────────┬──────────┬──────────┤
│ CPU仪表盘│ 内存仪表盘│ 系统雷达 │
└──────────┴──────────┴──────────┘
```

#### 2. 创建节点对比视图

**目标**: 对比多个节点的性能

**步骤**:
1. 为每个节点添加统计卡片
2. 使用相同的组件类型
3. 使用小尺寸以便并排显示

**布局建议**:
```
┌──────────┬──────────┬──────────┐
│ 节点1    │ 节点2    │ 节点3    │
│ 统计卡片 │ 统计卡片 │ 统计卡片 │
├──────────┼──────────┼──────────┤
│ 节点1    │ 节点2    │ 节点3    │
│ CPU仪表盘│ CPU仪表盘│ CPU仪表盘│
└──────────┴──────────┴──────────┘
```

#### 3. 创建专项监控视图

**目标**: 专注于某一类指标

**示例 - 网络监控**:
1. 添加多个网络趋势图
2. 每个图表监控不同节点
3. 使用大尺寸以便查看细节

**示例 - 存储监控**:
1. 添加多个磁盘饼图
2. 每个图表监控不同节点
3. 使用中尺寸以便对比

## 最佳实践

### 1. 布局设计

#### 原则
- **重要信息优先**: 将最重要的指标放在顶部
- **逻辑分组**: 相关的组件放在一起
- **适当留白**: 不要过度拥挤
- **一致性**: 使用统一的组件大小

#### 推荐布局模式

**模式1: 金字塔型**
```
┌─────────────────────────────────┐
│      统计卡片（大）              │  ← 概览
├──────────┬──────────┬──────────┤
│ 仪表盘   │ 仪表盘   │ 仪表盘   │  ← 关键指标
├──────────┴──────────┴──────────┤
│        趋势图（大）              │  ← 详细分析
└─────────────────────────────────┘
```

**模式2: 左右分栏**
```
┌──────────┬──────────────────────┐
│ 仪表盘   │                      │
├──────────┤   趋势图（大）       │
│ 仪表盘   │                      │
├──────────┼──────────────────────┤
│ 饼图     │   雷达图             │
└──────────┴──────────────────────┘
```

**模式3: 网格型**
```
┌──────────┬──────────┬──────────┐
│ 卡片1    │ 卡片2    │ 卡片3    │
├──────────┼──────────┼──────────┤
│ 卡片4    │ 卡片5    │ 卡片6    │
├──────────┼──────────┼──────────┤
│ 卡片7    │ 卡片8    │ 卡片9    │
└──────────┴──────────┴──────────┘
```

### 2. 组件选择

#### 根据场景选择

| 场景 | 推荐组件 | 原因 |
|------|---------|------|
| 快速概览 | 统计卡片 | 信息密度高 |
| 实时监控 | 仪表盘 | 直观易读 |
| 趋势分析 | 趋势图 | 显示变化 |
| 空间分析 | 饼图 | 占比清晰 |
| 综合评估 | 雷达图 | 多维对比 |

#### 根据节点数量选择

| 节点数 | 推荐方案 |
|--------|---------|
| 1-3个 | 使用大尺寸组件，详细展示 |
| 4-6个 | 使用中尺寸组件，平衡展示 |
| 7+个 | 使用小尺寸组件，概览展示 |

### 3. 性能优化

#### 建议
- 不要添加过多组件（建议≤12个）
- 优先使用统计卡片（性能最好）
- 大尺寸趋势图不要超过2个
- 定期清理不需要的组件

#### 性能对比

| 组件类型 | 性能 | 资源占用 |
|---------|------|---------|
| 统计卡片 | ⭐⭐⭐⭐⭐ | 低 |
| 仪表盘 | ⭐⭐⭐⭐ | 中 |
| 饼图 | ⭐⭐⭐⭐ | 中 |
| 趋势图 | ⭐⭐⭐ | 高 |
| 雷达图 | ⭐⭐⭐ | 高 |

## 常见问题

### Q1: 组件不显示数据怎么办？

**A**: 检查以下几点：
1. 确认选择的节点是否在线
2. 检查节点是否有数据上报
3. 刷新页面重试
4. 查看浏览器控制台是否有错误

### Q2: 布局保存失败？

**A**: 可能的原因：
1. 浏览器禁用了LocalStorage
2. 存储空间已满
3. 浏览器隐私模式

**解决方法**:
- 检查浏览器设置
- 清理浏览器缓存
- 使用普通模式

### Q3: 如何备份我的布局？

**A**: 当前版本布局保存在浏览器本地，建议：
1. 截图保存布局配置
2. 记录组件配置信息
3. 定期导出浏览器数据

### Q4: 可以创建多个仪表盘吗？

**A**: 当前版本仅支持单个仪表盘，后续版本将支持多仪表盘管理。

### Q5: 组件可以调整大小吗？

**A**: 当前版本组件大小在添加时固定，如需调整：
1. 删除原组件
2. 重新添加并选择新大小

### Q6: 如何分享我的仪表盘？

**A**: 当前版本不支持分享功能，建议：
1. 截图分享
2. 手动记录配置
3. 等待后续版本更新

### Q7: 移动端可以使用吗？

**A**: 可以，但建议：
- 使用横屏模式
- 减少组件数量
- 使用小尺寸组件

### Q8: 数据刷新频率是多少？

**A**: 
- 实时数据：1秒刷新
- 图表动画：平滑过渡
- 无需手动刷新
