# 快速开始指南

本指南帮助你快速搭建和运行分布式感知运维系统的开发环境。

---

## 📋 前置要求

### 必需软件

- **Node.js**: 18.x 或更高版本
- **Python**: 3.8 或更高版本
- **Redis**: 7.x 或更高版本
- **InfluxDB**: 2.x 或更高版本
- **Git**: 用于版本控制

### 可选软件

- **Docker**: 用于容器化部署
- **VS Code**: 推荐的开发工具

## 🚀 快速启动 (开发环境)

### 步骤 1: 克隆项目

```bash
git clone <repository-url>
cd project-root
```

### 步骤 2: 安装依赖

#### 后端依赖
```bash
cd backend
npm install
```

#### 前端依赖
```bash
cd frontend
npm install
```

#### Agent 依赖
```bash
cd agent
pip install -r requirements.txt
```

### 步骤 3: 配置环境变量

#### 后端配置
创建 `backend/.env` 文件:

```env
NODE_ENV=development
PORT=3000
WS_PORT=8080

# Redis (本地开发)
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# InfluxDB (本地开发)
INFLUX_URL=http://localhost:8086
INFLUX_TOKEN=your_token_here
INFLUX_ORG=my_org
INFLUX_BUCKET=system_monitoring

# JWT
JWT_SECRET=dev_secret_key_change_in_production
JWT_EXPIRES_IN=24h

# LLM (可选)
LLM_PROVIDER=openai
OPENAI_API_KEY=your_api_key_here
```

#### Agent 配置
编辑 `agent/config.yaml`:

```yaml
node:
  id: node001
  hostname: dev-machine

server:
  url: http://localhost:3000
  api_endpoint: /api/agent/metrics

collection:
  interval: 1
  retry: 3
  timeout: 5

logging:
  level: INFO
  file: ./agent.log
```

### 步骤 4: 启动数据库服务

#### 启动 Redis
```bash
# macOS (Homebrew)
brew services start redis

# Linux (systemd)
sudo systemctl start redis

# Windows
redis-server
```

#### 启动 InfluxDB
```bash
# macOS (Homebrew)
brew services start influxdb

# Linux (systemd)
sudo systemctl start influxdb

# Windows
influxd
```

#### 配置 InfluxDB
访问 http://localhost:8086 完成初始化:
1. 创建组织 (Organization)
2. 创建 Bucket: `system_monitoring`
3. 生成 API Token
4. 将 Token 复制到 `.env` 文件

### 步骤 5: 启动服务

#### 启动后端 (终端 1)
```bash
cd backend
npm run dev
```

#### 启动前端 (终端 2)
```bash
cd frontend
npm run dev
```

#### 启动 Agent (终端 3)
```bash
cd agent
python agent.py
```

### 步骤 6: 访问应用

- **前端界面**: http://localhost:5174
- **后端 API**: http://localhost:3000
- **WebSocket**: ws://localhost:8080

**默认登录账号**:
- 用户名: `admin`
- 密码: `admin`

## 🐳 Docker 快速启动 (推荐)

### 步骤 1: 安装 Docker

确保已安装 Docker 和 Docker Compose。

### 步骤 2: 启动所有服务

```bash
# 在项目根目录执行
docker-compose up -d
```

这将启动:
- Redis
- InfluxDB
- 后端服务
- 前端服务

### 步骤 3: 查看服务状态

```bash
docker-compose ps
```

### 步骤 4: 查看日志

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
```

### 步骤 5: 停止服务

```bash
docker-compose down
```

## 🔧 常见问题

### Q1: Redis 连接失败

**问题**: `Error: connect ECONNREFUSED 127.0.0.1:6379`

**解决方案**:
1. 确认 Redis 服务已启动
2. 检查 Redis 端口是否正确
3. 检查防火墙设置

```bash
# 检查 Redis 状态
redis-cli ping
# 应返回: PONG
```

### Q2: InfluxDB 连接失败

**问题**: `Error: unauthorized access`

**解决方案**:
1. 确认 InfluxDB Token 正确
2. 确认 Organization 和 Bucket 名称正确
3. 重新生成 Token

### Q3: 前端无法连接后端

**问题**: `Network Error` 或 CORS 错误

**解决方案**:
1. 确认后端服务已启动
2. 检查 `frontend/.env` 中的 API 地址
3. 检查后端 CORS 配置

### Q4: Agent 无法上报数据

**问题**: Agent 运行但数据未显示

**解决方案**:
1. 检查 Agent 配置文件中的服务器地址
2. 查看 Agent 日志文件
3. 确认后端接口正常工作

```bash
# 测试后端接口
curl -X POST http://localhost:3000/api/agent/metrics \
  -H "Content-Type: application/json" \
  -d '{"node_id":"test","metrics":{"cpu_usage":50},"timestamp":"2026-02-06T10:00:00Z"}'
```
