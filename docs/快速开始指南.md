# 快速开始指南

本指南用于快速完成 EOMS 的本地联调，内容已按当前代码与配置文件更新。

---

## 前置要求

必需软件：

- Node.js 18+
- Python 3.8+
- Redis 7+，推荐
- InfluxDB 2+，推荐

当前默认地址：

- 前端：`http://localhost:5174`
- 后端：`http://localhost:50001`
- WebSocket：`ws://localhost:50001/ws/metrics`

---

## 第一步：安装依赖

后端：

```bash
cd backend
npm install
```

前端：

```bash
cd frontend
npm install
```

Agent：

```bash
cd agent
pip install -r requirements.txt
```

---

## 第二步：配置后端

复制环境变量模板：

```bash
cd backend
cp .env.example .env
```

至少确认以下字段：

```env
NODE_ENV=development
PORT=50001
HOST=0.0.0.0

JWT_SECRET=your_jwt_secret_key_change_in_production
AGENT_API_TOKEN=change_me_agent_token
ALLOW_DEFAULT_ADMIN_PASSWORD=false

REDIS_HOST=localhost
REDIS_PORT=6379

INFLUX_URL=http://localhost:8086
INFLUX_TOKEN=your_influx_token_here
INFLUX_ORG=eoms
INFLUX_BUCKET=system_monitoring
```

说明：

- 不配置 `JWT_SECRET`，鉴权能力无法安全使用
- 不配置 `AGENT_API_TOKEN`，Agent 上报会被拒绝

---

## 第三步：配置 Agent

编辑 [config.yaml](/C:/Users/ASUS/Desktop/01_学习资料/毕设/EOMS/agent/config.yaml)：

```yaml
node:
  id: node001
  hostname: dev-machine
  display_name: dev-machine

server:
  url: http://localhost:50001
  register_endpoint: /api/agent/register
  metrics_endpoint: /api/agent/metrics
  agent_token: CHANGE_ME_AGENT_TOKEN

collection:
  interval: 1
  retry: 3
  timeout: 5
```

要求：

- `server.url` 指向后端 `50001`
- `agent_token` 必须与后端 `AGENT_API_TOKEN` 完全一致

---

## 第四步：启动基础依赖

Redis：

```bash
# macOS
brew services start redis

# Linux
sudo systemctl start redis

# Windows
redis-server
```

InfluxDB：

```bash
# macOS
brew services start influxdb

# Linux
sudo systemctl start influxdb

# Windows
influxd
```

如首次使用 InfluxDB，请访问 `http://localhost:8086` 初始化并生成 Token。

---

## 第五步：启动系统

终端 1，启动后端：

```bash
cd backend
npm run dev
```

终端 2，启动前端：

```bash
cd frontend
npm run dev
```

终端 3，启动 Agent：

```bash
cd agent
python agent.py
```

---

## 第六步：访问系统

- 前端界面：`http://localhost:5174`
- 后端健康检查：`http://localhost:50001/health`

默认管理员账号是否存在，取决于本地初始化和环境变量策略。若系统初始化了默认管理员，请在首次登录后立即修改密码。

## 常见问题

### 1. 前端连不上后端

检查：

- 后端是否运行在 `50001`
- 前端接口地址是否仍指向旧端口
- 防火墙是否放行

### 2. Agent 无法注册或上报

检查：

- `agent/config.yaml` 中 `server.url`
- `agent_token` 与后端 `AGENT_API_TOKEN` 是否一致
- 后端日志是否有 token reject 记录

### 3. InfluxDB 连接失败

检查：

- `INFLUX_URL`
- `INFLUX_TOKEN`
- `INFLUX_ORG`
- `INFLUX_BUCKET`
