# EOMS 文档收敛方案（2026-04-17）

## 目标

- 消除文档口径冲突（端口、发布状态、完成度）
- 建立单一事实源（SSOT）
- 降低维护成本，保留必要历史记录

## 现状问题

- 发布状态冲突：同仓库同时存在“发布前收口”和“已具备生产部署能力”两种表述
- 端口冲突：`50001` 与 `3000` 并存
- 阶段冲突：部分文档仍停留在“AI/告警未实现”，与现有代码不符
- 文档数量多且重复，阅读入口不统一

## 单一事实源（SSOT）定义

- 一级（对外口径）：
  - 仓库根目录 `README.md`
  - `docs/README.md`
- 二级（执行手册）：
  - `docs/START.md`
  - `docs/部署指南.md`
  - `docs/测试指南.md`
- 三级（设计与决策）：
  - `docs/系统设计方案.md`
  - `docs/技术实施细节.md`
  - `docs/adr/*`
- 历史归档（不再作为当前事实来源）：
  - 各类“完成报告/修复报告/阶段总结”

## 一次性修订计划（建议 1 天完成）

1. 统一口径
- 以 `README.md` 为准，统一端口为 `50001`
- 所有文档统一声明“当前为发布前工程化收口，不宣称生产可用”

2. 入口收敛
- `docs/README.md` 仅保留 8-10 个核心入口
- 对历史文档增加“历史记录，不代表当前状态”提示

3. 历史文档分层
- 将历史阶段报告移动到 `docs/archive/`
- 保留文件名，不改内容，避免丢失追溯链

4. 冲突清理
- 重点修订：`开发进度.md`、`当前状态总结.md`、`部署指南.md`
- 修订后运行一次人工抽查：随机抽 5 份文档校验端口/状态口径

5. 维护机制
- 后续每次大功能合并时，仅允许更新 SSOT 文档
- 其他文档引用 SSOT，不重复写“当前状态”

## 验收标准

- 任意文档检索 `50001` / `3000` 时，不再出现并存冲突（历史归档除外）
- 任意文档检索“生产可用”时，不出现与 SSOT 冲突描述
- 新人按 `docs/README.md` 可在 15 分钟内找到启动、测试、部署入口
