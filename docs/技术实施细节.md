## 架构概览

EOMS 由四部分组成：

- `agent/`：Python 采集端
- `backend/`：Node.js + Express API 服务
- `frontend/`：Vue 3 + TypeScript 控制台
- `electron/`：桌面壳

核心数据流：

```text
Agent -> /api/agent/register
Agent -> /api/agent/metrics
Backend -> Redis latest cache
Backend -> InfluxDB history store
Backend -> WebSocket /ws/metrics
Frontend <- API + WebSocket
```

## 认证机制

### 1. 用户认证

用户接口主要使用 JWT。

典型流程：

1. `POST /api/auth/login`
2. 返回 `data.token`
3. 前端后续通过 `Authorization: Bearer <token>` 访问受保护接口

登录接口实现见 [auth.js](/C:/Users/ASUS/Desktop/01_学习资料/毕设/EOMS/backend/routes/auth.js)。

### 2. Agent 认证

Agent 注册和指标上报使用独立 Token：

- Header：`X-Agent-Token`
- 后端环境变量：`AGENT_API_TOKEN`
- Agent 配置字段：`server.agent_token`

实现见 [agent.js](/C:/Users/ASUS/Desktop/01_学习资料/毕设/EOMS/backend/routes/agent.js)。

### 3. WebSocket 认证

WebSocket 通过 JWT 认证。

支持：

- Query 参数：`?token=<jwt>`
- 或 `Authorization: Bearer <token>`

实现见 [metricsWS.js](/C:/Users/ASUS/Desktop/01_学习资料/毕设/EOMS/backend/websocket/metricsWS.js)。

## 关键接口

以下接口为当前代码中最稳定、最常用的一组。

### 1. 健康检查

```http
GET /health
```

返回：

- `status`
- `timestamp`
- `uptime`
- `redis`
- `websocket`

### 2. 登录

```http
POST /api/auth/login
```

请求体：

```json
{
  "username": "admin",
  "password": "admin"
}
```

成功响应关键字段：

```json
{
  "success": true,
  "data": {
    "token": "jwt-token",
    "user": {
      "id": "user-id",
      "username": "admin",
      "role": "admin"
    }
  }
}
```

### 3. Agent 注册

```http
POST /api/agent/register
X-Agent-Token: <agent-token>
```

请求体关键字段：

```json
{
  "node_id": "node001",
  "hostname": "server-01",
  "ip": "192.168.1.100",
  "display_name": "Node 001"
}
```

### 4. Agent 上报指标

```http
POST /api/agent/metrics
X-Agent-Token: <agent-token>
```

请求体关键字段：

```json
{
  "node_id": "node001",
  "metrics": {
    "cpu_usage": 45.6,
    "memory_usage": 78.2,
    "disk_usage": 65.3,
    "network_rx_bytes": 1024000,
    "network_tx_bytes": 512000
  },
  "timestamp": "2026-04-17T10:30:00Z"
}
```

当前后端要求 `metrics` 中至少包含：

- `cpu_usage`
- `memory_usage`
- `disk_usage`

### 5. 获取最新数据

```http
GET /api/metrics/latest/:nodeId
```

当前返回结构：

```json
{
  "success": true,
  "data": {
    "node_id": "node001",
    "metrics": {
      "cpu_usage": 45.6
    },
    "timestamp": "2026-04-17T10:30:00Z"
  }
}
```

### 6. 获取历史数据

```http
GET /api/metrics/history?nodeId=...&startTime=...&endTime=...&interval=...
```

当前必填参数：

- `nodeId`
- `startTime`
- `endTime`

可选参数：

- `interval`，默认 `1m`

### 7. 业务扩展接口

当前代码已具备以下模块接口：

- 告警：`/api/alert`
- AI 分析：`/api/ai`
- AI Ops：`/api/ai/v2`
- 用户管理：`/api/users`
- 报表：`/api/reports`

这些模块已有实际路由与服务实现，不应在文档中继续描述为“未开始”。

## WebSocket 协议

### 连接地址

```text
ws://localhost:50001/ws/metrics?token=<jwt>
```

### 连接成功消息

```json
{
  "type": "connected",
  "clientId": "client_xxx",
  "message": "Connected to EOMS WebSocket Server"
}
```

### 订阅节点

客户端发送：

```json
{
  "type": "subscribe",
  "node_id": "node001"
}
```

服务端返回：

```json
{
  "type": "subscribed",
  "node_id": "node001"
}
```

### 订阅全部节点

客户端发送：

```json
{
  "type": "subscribe_all"
}
```

### 心跳

客户端：

```json
{
  "type": "ping"
}
```

服务端：

```json
{
  "type": "pong"
}
```

### 指标推送

服务端广播示例：

```json
{
  "type": "metrics",
  "node_id": "node001",
  "data": {
    "cpu_usage": 45.6,
    "memory_usage": 78.2,
    "disk_usage": 65.3
  },
  "timestamp": "2026-04-17T10:30:00Z"
}
```

## 关键配置

### 后端 `.env`

当前建议最少配置：

```env
NODE_ENV=development
PORT=50001
HOST=0.0.0.0
JWT_SECRET=your_jwt_secret_key_change_in_production
AGENT_API_TOKEN=change_me_agent_token
ALLOW_DEFAULT_ADMIN_PASSWORD=false
REDIS_HOST=localhost
REDIS_PORT=6379
INFLUX_URL=http://localhost:8086
INFLUX_TOKEN=your_influx_token_here
INFLUX_ORG=eoms
INFLUX_BUCKET=system_monitoring
```

### Agent `config.yaml`

```yaml
node:
  id: node001
  hostname: dev-machine
  display_name: dev-machine

server:
  url: http://localhost:50001
  register_endpoint: /api/agent/register
  metrics_endpoint: /api/agent/metrics
  agent_token: CHANGE_ME_AGENT_TOKEN

collection:
  interval: 1
  retry: 3
  timeout: 5
```
