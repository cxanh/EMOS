# 📧 告警通知配置指南

**版本**: 1.0  
**日期**: 2026-02-09

---

## 📋 概述

EOMS 告警系统支持三种通知方式：
- ✅ **WebSocket 推送** - 默认启用，无需配置
- 📧 **邮件通知** - 需要配置 SMTP 服务器
- 💬 **钉钉通知** - 需要配置钉钉机器人

---

## 🚀 快速开始

### 1. 安装依赖

```bash
cd backend
npm install
```

这会自动安装 `nodemailer` 和 `axios` 依赖。

### 2. 配置通知渠道

编辑 `backend/.env` 文件，添加相应的配置。

## 📧 邮件通知配置

### Gmail 配置（推荐）

#### 步骤 1: 启用两步验证

1. 访问 Google 账户设置: https://myaccount.google.com/security
2. 找到"两步验证"并启用

#### 步骤 2: 生成应用专用密码

1. 在"两步验证"页面，找到"应用专用密码"
2. 选择"邮件"和"Windows 计算机"
3. 点击"生成"
4. 复制生成的 16 位密码

#### 步骤 3: 配置 .env

在 `backend/.env` 文件中添加：

```env
# 邮件通知配置
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-16-digit-app-password
ALERT_EMAIL=admin@example.com
```

**配置说明**:
- `SMTP_HOST`: SMTP 服务器地址
- `SMTP_PORT`: SMTP 端口（587 for TLS, 465 for SSL）
- `SMTP_SECURE`: 是否使用 SSL（587端口用false，465端口用true）
- `SMTP_USER`: 发件人邮箱
- `SMTP_PASS`: 应用专用密码（不是邮箱密码！）
- `ALERT_EMAIL`: 接收告警的邮箱（可选，默认使用 SMTP_USER）

### QQ 邮箱配置

#### 步骤 1: 开启 SMTP 服务

1. 登录 QQ 邮箱
2. 设置 → 账户
3. 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"
4. 开启"IMAP/SMTP服务"
5. 生成授权码

#### 步骤 2: 配置 .env

```env
# 邮件通知配置
SMTP_HOST=smtp.qq.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your-qq-email@qq.com
SMTP_PASS=your-authorization-code
ALERT_EMAIL=admin@example.com
```

### 163 邮箱配置

#### 步骤 1: 开启 SMTP 服务

1. 登录 163 邮箱
2. 设置 → POP3/SMTP/IMAP
3. 开启"IMAP/SMTP服务"
4. 设置客户端授权密码

#### 步骤 2: 配置 .env

```env
# 邮件通知配置
SMTP_HOST=smtp.163.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your-email@163.com
SMTP_PASS=your-authorization-password
ALERT_EMAIL=admin@example.com
```

### 企业邮箱配置

根据你的企业邮箱提供商，配置相应的 SMTP 服务器：

```env
# 阿里云企业邮箱

SMTP_PORT=465
SMTP_SECURE=true

# 腾讯企业邮箱

SMTP_PORT=465
SMTP_SECURE=true

# Microsoft 365

SMTP_PORT=587
SMTP_SECURE=false
```

## 💬 钉钉通知配置

### 步骤 1: 创建钉钉群机器人

1. 打开钉钉，进入要接收告警的群聊
2. 点击右上角"..."→"群设置"
3. 点击"智能群助手"→"添加机器人"
4. 选择"自定义"机器人
5. 设置机器人名称（如"EOMS告警"）
6. 安全设置选择"加签"（推荐）
7. 复制 Webhook 地址和加签密钥

### 步骤 2: 配置 .env

在 `backend/.env` 文件中添加：

```env
# 钉钉通知配置
DINGTALK_WEBHOOK=https://oapi.dingtalk.com/robot/send?access_token=YOUR_ACCESS_TOKEN
DINGTALK_SECRET=SECxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**配置说明**:
- `DINGTALK_WEBHOOK`: 机器人的 Webhook 地址
- `DINGTALK_SECRET`: 加签密钥（如果选择了加签安全设置）

### 钉钉安全设置说明

钉钉机器人有三种安全设置：

1. **自定义关键词**: 消息中必须包含设置的关键词
   - 设置关键词如"告警"、"监控"
   - 不需要配置 `DINGTALK_SECRET`

2. **加签** (推荐): 使用密钥签名验证
   - 需要配置 `DINGTALK_SECRET`
   - 更安全

3. **IP地址**: 限制只能从指定IP发送
   - 适合固定IP的服务器
   - 不需要额外配置

## 🧪 测试配置

### 测试邮件通知

创建测试脚本 `backend/test-email.js`:

```javascript
require('dotenv').config();
const emailService = require('./services/emailService');

async function test() {
  await emailService.initialize();
  
  if (!emailService.isEnabled()) {
    console.log('❌ Email service is not enabled');
    return;
  }
  
  const testEvent = {
    ruleName: '测试告警',
    nodeId: 'test-node',
    nodeName: '测试节点',
    metric: 'cpu_usage',
    currentValue: 85.5,
    threshold: 80,
    triggeredAt: new Date().toISOString(),
    message: '这是一条测试告警消息'
  };
  
  const result = await emailService.sendAlert(testEvent);
  
  if (result.success) {
    console.log('✅ Email sent successfully!');
  } else {
    console.log('❌ Failed to send email:', result.error);
  }
}

test();
```

运行测试:
```bash
node backend/test-email.js
```

## 🎯 使用告警通知

### 创建规则时选择通知方式

1. 登录系统
2. 进入"告警管理"页面
3. 点击"新建规则"
4. 在"通知方式"中勾选需要的通知渠道：
   - ✅ WebSocket推送（实时前端通知）
   - ✅ 邮件通知（需要配置）
   - ✅ 钉钉通知（需要配置）
5. 创建规则

### 通知触发时机

- **告警触发**: 当指标超过阈值并持续指定时间后
- **告警恢复**: 当指标恢复正常后

### 通知内容

#### 邮件通知内容

- 告警规则名称
- 节点信息
- 监控指标
- 当前值（红色高亮）
- 阈值
- 触发时间
- 告警消息
- 系统登录链接

#### 钉钉通知内容

- 告警规则名称
- 节点信息
- 监控指标
- 当前值（红色标注）
- 阈值
- 触发时间
- 告警消息

## 🐛 常见问题

### 邮件通知问题

#### 问题 1: 邮件发送失败 - 认证错误

**错误信息**: `Invalid login: 535 Authentication failed`

**解决方法**:
1. 确认使用的是应用专用密码，不是邮箱登录密码
2. 检查邮箱是否开启了 SMTP 服务
3. 确认 SMTP_USER 和 SMTP_PASS 配置正确

#### 问题 2: 邮件发送失败 - 连接超时

**错误信息**: `Connection timeout`

**解决方法**:
1. 检查网络连接
2. 确认 SMTP_HOST 和 SMTP_PORT 正确
3. 检查防火墙是否阻止了 SMTP 端口
4. 尝试使用不同的端口（587 或 465）

#### 问题 3: Gmail 提示"不够安全的应用"

**解决方法**:
1. 必须启用两步验证
2. 使用应用专用密码，不要使用账户密码
3. 不要开启"允许不够安全的应用"

### 钉钉通知问题

#### 问题 1: 钉钉消息发送失败 - 签名错误

**错误信息**: `sign not match`

**解决方法**:
1. 确认 DINGTALK_SECRET 配置正确
2. 检查是否复制了完整的密钥（包括 SEC 前缀）
3. 确认机器人安全设置选择的是"加签"

#### 问题 2: 钉钉消息发送失败 - 关键词不匹配

**错误信息**: `keywords not in content`

**解决方法**:
1. 如果使用"自定义关键词"安全设置，消息必须包含关键词
2. 建议改用"加签"安全设置
3. 或者在机器人设置中添加"告警"、"监控"等关键词

#### 问题 3: 钉钉消息发送失败 - IP不在白名单

**错误信息**: `ip not in whitelist`

**解决方法**:
1. 在钉钉机器人设置中添加服务器IP到白名单
2. 或者改用"加签"安全设置

## 📊 通知状态检查

### 查看后端日志

启动后端时会显示通知服务状态：

```
Email Service Initialized          # 邮件服务已启用
DingTalk Service Initialized       # 钉钉服务已启用
```

或者：

```
Email service not configured       # 邮件服务未配置
DingTalk service not configured    # 钉钉服务未配置
```

### 查看告警日志

当告警触发时，日志会显示：

```
Alert triggered: CPU使用率过高 on node001 (85.5% > 80%)
Alert email sent: <message-id>
Alert sent to DingTalk successfully
Alert notification sent: event_xxx
```

1. 创建 `backend/services/xxxService.js`
2. 实现 `initialize()`, `sendAlert()`, `sendRecovery()` 方法
3. 在 `backend/index.js` 中导入并初始化
4. 在 `backend/services/alertChecker.js` 中添加通知逻辑
5. 在前端添加通知渠道选项
