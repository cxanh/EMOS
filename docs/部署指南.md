# EOMS 系统部署指南

本指南以当前代码和配置文件为准，覆盖本地联调、服务器部署和容器化参考方案。

**文档口径说明**：

- 当前默认后端端口为 `50001`
- 当前项目支持本地联调和部署准备
- 当前项目仍处于发布前工程化收口阶段，不应直接宣称“已经可以直接按生产环境发布”

### 最低配置

- CPU：2 核
- 内存：4 GB

- Node.js 18+
- Python 3.8+
- Redis 7+，推荐
- InfluxDB 2+，推荐

## 开发环境部署

### 1. 获取代码

```bash
git clone <repository-url>
cd EOMS
```

### 2. 安装依赖

后端：

```bash
cd backend
npm install
```

前端：

```bash
cd frontend
npm install
```

Agent：

```bash
cd agent
pip install -r requirements.txt
```

### 3. 配置环境变量

#### 后端配置

复制 `backend/.env.example` 为 `backend/.env`：

```bash
cd backend
cp .env.example .env
```

最少需要检查和配置以下字段：

```env
NODE_ENV=development
PORT=50001
HOST=0.0.0.0

JWT_SECRET=your_jwt_secret_key_change_in_production
AGENT_API_TOKEN=change_me_agent_token
ALLOW_DEFAULT_ADMIN_PASSWORD=false

REDIS_HOST=localhost
REDIS_PORT=6379

INFLUX_URL=http://localhost:8086
INFLUX_TOKEN=your_influx_token_here
INFLUX_ORG=eoms
INFLUX_BUCKET=system_monitoring
```

说明：

- `JWT_SECRET` 未设置时，认证能力不可安全使用
- `AGENT_API_TOKEN` 未设置时，Agent 请求会被拒绝
- `ALLOW_DEFAULT_ADMIN_PASSWORD=false` 建议保持关闭

#### Agent 配置

编辑 [config.yaml](/C:/Users/ASUS/Desktop/01_学习资料/毕设/EOMS/agent/config.yaml)：

```yaml
node:
  id: node001
  hostname: dev-machine
  display_name: dev-machine

server:
  url: http://localhost:50001
  register_endpoint: /api/agent/register
  metrics_endpoint: /api/agent/metrics
  agent_token: CHANGE_ME_AGENT_TOKEN

collection:
  interval: 1
  retry: 3
  timeout: 5
```

要求：

- `server.url` 指向后端地址
- `agent_token` 必须与后端 `AGENT_API_TOKEN` 保持一致

## 服务器部署

### 1. 构建前端

```bash
cd frontend
npm run build
```

构建产物位于 `frontend/dist`。

### 2. Nginx 反向代理

参考配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /var/www/eoms/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:50001/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /ws/ {
        proxy_pass http://127.0.0.1:50001/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/eoms /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3. 使用 PM2 管理后端

安装 PM2：

```bash
npm install -g pm2
```

示例 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [
    {
      name: 'eoms-backend',
      script: './backend/index.js',
      instances: 2,
      exec_mode: 'cluster',
      env: {
        NODE_ENV: 'production',
        PORT: 50001,
        HOST: '0.0.0.0'
      },
      error_file: './logs/backend-error.log',
      out_file: './logs/backend-out.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss'
    }
  ]
};
```

启动：

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

### 4. 将 Agent 配置为系统服务

#### Linux systemd

```ini
[Unit]
Description=EOMS Agent Service
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/EOMS/agent
ExecStart=/usr/bin/python3 /path/to/EOMS/agent/agent.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用：

```bash
sudo systemctl daemon-reload
sudo systemctl enable eoms-agent
sudo systemctl start eoms-agent
sudo systemctl status eoms-agent
```

#### Windows NSSM

```cmd
nssm install EOMSAgent
```

配置：

- Path：`C:\Python\python.exe`
- Startup directory：`C:\path\to\EOMS\agent`
- Arguments：`agent.py`

启动：

```cmd
nssm start EOMSAgent
```

### 5. 防火墙建议

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 50001/tcp
```

说明：

- 若后端仅通过 Nginx 内网反代，可不直接开放 `50001`
- 若 Agent 来自其他机器，需要保证 Agent 到后端的网络可达

## Docker 部署参考

当前仓库并未提供现成、已验证的 Docker 部署产物。以下内容仅作为参考模板，使用前需要结合当前代码继续完善与验证。

### 后端 Dockerfile

```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 50001

CMD ["node", "index.js"]
```

### 前端 Dockerfile

```dockerfile
FROM node:18-alpine AS build

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### Agent Dockerfile

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "agent.py"]
```

### docker-compose 参考

```yaml
version: '3.8'

services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    restart: unless-stopped

  influxdb:
    image: influxdb:2.7-alpine
    ports:
      - "8086:8086"
    restart: unless-stopped

  backend:
    build: ./backend
    ports:
      - "50001:50001"
    environment:
      - NODE_ENV=production
      - PORT=50001
      - HOST=0.0.0.0
      - REDIS_HOST=redis
      - INFLUX_URL=http://influxdb:8086
    depends_on:
      - redis
      - influxdb
    restart: unless-stopped

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

  agent:
    build: ./agent
    environment:
      - SERVER_URL=http://backend:50001
    depends_on:
      - backend
    restart: unless-stopped
```

注意：

- 该 Compose 方案仍需要结合 `AGENT_API_TOKEN`、JWT、数据卷和日志目录继续补全
- 在未完成联调前，不建议将其作为正式生产方案

## 常见问题

### 1. Agent 上报被拒绝

可能原因：

- 后端未配置 `AGENT_API_TOKEN`
- Agent 配置中的 `agent_token` 与后端不一致

排查方式：

- 检查 `backend/.env`
- 检查 `agent/config.yaml`
- 查看后端日志中是否出现 token reject 记录

### 2. 前端无法访问后端

排查方式：

- 确认后端已启动在 `50001`
- 检查前端环境变量或代理配置
- 检查 Nginx 转发配置
- 检查防火墙

### 3. InfluxDB 连接失败

排查方式：

1. 访问 `http://localhost:8086`
2. 确认 Token、Org、Bucket 正确
3. 检查 `.env` 中配置是否为真实值

### 4. PM2 进程异常重启

排查方式：

```bash
pm2 logs eoms-backend
pm2 status
lsof -i :50001
```

### 5. Nginx 502

排查方式：

```bash
sudo nginx -t
pm2 status
curl http://127.0.0.1:50001/health
```
