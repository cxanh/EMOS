## 系统要求

### 最低配置
- **CPU**: 2核
- **内存**: 4GB
- **磁盘**: 20GB
- **操作系统**: Windows 10+, Ubuntu 18.04+, macOS 10.15+

### 软件依赖
- **Node.js**: 18.0.0 或更高
- **Python**: 3.8 或更高
- **Redis**: 7.0 或更高 (可选)
- **InfluxDB**: 2.0 或更高 (可选)

## 开发环境部署

### 1. 克隆项目

```bash
git clone <repository-url>
cd eoms-system
```

### 2. 安装依赖

#### 后端依赖
```bash
cd backend
npm install
```

#### 前端依赖
```bash
cd frontend
npm install
```

#### Agent 依赖
```bash
cd agent
pip install -r requirements.txt
```

### 3. 配置环境变量

#### 后端配置
复制 `backend/.env.example` 为 `backend/.env`:

```bash
cd backend
cp .env.example .env
```

编辑 `.env` 文件：

```env
# 服务配置
NODE_ENV=development
PORT=3000
WS_PORT=8080

# Redis 配置 (可选)
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# InfluxDB 配置 (可选)
INFLUX_URL=http://localhost:8086
INFLUX_TOKEN=your_token_here
INFLUX_ORG=eoms
INFLUX_BUCKET=system_monitoring

# JWT 配置
JWT_SECRET=your_secret_key_here
JWT_EXPIRES_IN=24h
```

#### Agent 配置
编辑 `agent/config.yaml`:

```yaml
node:
  id: node001
  hostname: dev-machine

server:
  url: http://localhost:3000
  register_endpoint: /api/agent/register
  metrics_endpoint: /api/agent/metrics

collection:
  interval: 1
  retry: 3
  timeout: 5

logging:
  level: INFO
  file: agent.log
```

### 4. 启动服务

#### 启动 Redis (可选)
```bash
# macOS
brew services start redis

# Linux
sudo systemctl start redis

# Windows
redis-server
```

#### 启动 InfluxDB (可选)
```bash
# macOS
brew services start influxdb

# Linux
sudo systemctl start influxdb

# Windows
influxd
```

#### 启动后端
```bash
cd backend
npm run dev
```

#### 启动前端
```bash
cd frontend
npm run dev
```

#### 启动 Agent
```bash
cd agent
python agent.py
```

### 5. 访问系统

- **前端**: http://localhost:5174
- **后端 API**: http://localhost:3000
- **默认账号**: admin / admin

## 生产环境部署

### 1. 构建前端

```bash
cd frontend
npm run build
```

构建产物在 `frontend/dist` 目录。

### 2. 配置 Nginx

创建 Nginx 配置文件 `/etc/nginx/sites-available/eoms`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/eoms/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # WebSocket
    location /ws {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/eoms /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3. 使用 PM2 管理后端进程

安装 PM2:

```bash
npm install -g pm2
```

创建 PM2 配置文件 `ecosystem.config.js`:

```javascript
module.exports = {
  apps: [{
    name: 'eoms-backend',
    script: './backend/index.js',
    instances: 2,
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    error_file: './logs/backend-error.log',
    out_file: './logs/backend-out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss'
  }]
};
```

启动服务：

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

### 4. 配置 Agent 为系统服务

#### Linux (systemd)

创建服务文件 `/etc/systemd/system/eoms-agent.service`:

```ini
[Unit]
Description=EOMS Agent Service
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/agent
ExecStart=/usr/bin/python3 /path/to/agent/agent.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable eoms-agent
sudo systemctl start eoms-agent
sudo systemctl status eoms-agent
```

#### Windows (NSSM)

下载 NSSM: https://nssm.cc/download

```cmd
nssm install EOMSAgent
# Path: C:\Python\python.exe
# Startup directory: C:\path\to\agent
# Arguments: agent.py

nssm start EOMSAgent
```

### 5. 配置防火墙

```bash
# 允许 HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 允许后端端口 (如果需要外部访问)
sudo ufw allow 3000/tcp
```

## Docker 部署

### 1. 创建 Docker 镜像

#### 后端 Dockerfile

创建 `backend/Dockerfile`:

```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 3000

CMD ["node", "index.js"]
```

#### 前端 Dockerfile

创建 `frontend/Dockerfile`:

```dockerfile
FROM node:18-alpine as build

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

#### Agent Dockerfile

创建 `agent/Dockerfile`:

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "agent.py"]
```

### 2. Docker Compose

创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    restart: unless-stopped

  influxdb:
    image: influxdb:2.7-alpine
    ports:
      - "8086:8086"
    volumes:
      - influxdb-data:/var/lib/influxdb2
    environment:
      - DOCKER_INFLUXDB_INIT_MODE=setup
      - DOCKER_INFLUXDB_INIT_USERNAME=admin
      - DOCKER_INFLUXDB_INIT_PASSWORD=adminpassword
      - DOCKER_INFLUXDB_INIT_ORG=eoms
      - DOCKER_INFLUXDB_INIT_BUCKET=system_monitoring
    restart: unless-stopped

  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - REDIS_HOST=redis
      - INFLUX_URL=http://influxdb:8086
    depends_on:
      - redis
      - influxdb
    restart: unless-stopped

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

  agent:
    build: ./agent
    environment:
      - SERVER_URL=http://backend:3000
    depends_on:
      - backend
    restart: unless-stopped

volumes:
  redis-data:
  influxdb-data:
```

### 3. 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 停止并删除数据
docker-compose down -v
```

## 常见问题

### Q1: Redis 连接失败

**问题**: `Error: connect ECONNREFUSED 127.0.0.1:6379`

**解决方案**:
1. 确认 Redis 服务已启动
2. 检查 Redis 端口是否正确
3. 检查防火墙设置
4. 如果不需要 Redis，可以在代码中禁用

### Q2: InfluxDB Token 配置

**问题**: InfluxDB 连接失败

**解决方案**:
1. 访问 http://localhost:8086
2. 完成初始化设置
3. 创建 Token: Settings → Tokens → Generate Token
4. 将 Token 复制到 `.env` 文件

### Q3: 前端无法连接后端

**问题**: Network Error 或 CORS 错误

**解决方案**:
1. 确认后端服务已启动
2. 检查 `frontend/.env` 中的 API 地址
3. 检查后端 CORS 配置
4. 检查防火墙设置

### Q4: Agent 无法上报数据

**问题**: Agent 运行但数据未显示

**解决方案**:
1. 检查 Agent 配置文件中的服务器地址
2. 查看 Agent 日志文件
3. 确认后端接口正常工作
4. 检查网络连接

### Q5: PM2 进程崩溃

**问题**: 后端进程频繁重启

**解决方案**:
1. 查看错误日志: `pm2 logs eoms-backend`
2. 检查端口占用: `lsof -i :3000`
3. 检查内存使用: `pm2 monit`
4. 增加重启延迟: `restart_delay: 5000`

### Q6: Nginx 502 错误

**问题**: Bad Gateway

**解决方案**:
1. 确认后端服务运行: `pm2 status`
2. 检查 Nginx 配置: `sudo nginx -t`
3. 查看 Nginx 日志: `sudo tail -f /var/log/nginx/error.log`
4. 检查 SELinux 设置 (Linux)
