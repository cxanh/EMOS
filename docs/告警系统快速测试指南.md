## 🎯 测试步骤

### 步骤 1: 启动后端服务

```bash
cd backend
npm run dev
```

**预期输出**:
```
Redis Client Connected
Redis Client Ready
InfluxDB Client Connected
Data Store Service Initialized
Alert Service Initialized
Alert checker started
EOMS Backend Server running on http://127.0.0.1:50001
```

### 步骤 2: 访问前端

1. 打开浏览器访问: http://localhost:5174
2. 登录系统（用户名: admin, 密码: admin123）
3. 点击侧边栏"告警管理"菜单

### 步骤 3: 创建测试规则

1. 点击"新建规则"按钮
2. 填写规则信息:
   - **规则名称**: CPU使用率测试
   - **监控节点**: 所有节点
   - **监控指标**: CPU使用率
   - **条件**: 大于 (>)
   - **阈值**: 50 (设置较低的值便于触发)
   - **持续时间**: 10 (秒)
   - **通知方式**: 勾选 WebSocket推送
3. 点击"创建"

### 步骤 4: 触发告警

**方法 1: 运行 CPU 密集型任务**

Windows PowerShell:
```powershell
# 运行此命令会占用大量 CPU
while($true) { 
    $result = 1
    for($i=0; $i -lt 10000000; $i++) { 
        $result = $result * 2 
    } 
}
```

按 `Ctrl+C` 停止

**方法 2: 降低阈值**

如果当前 CPU 使用率是 60%:
1. 编辑刚创建的规则
2. 将阈值改为 40
3. 保存

### 步骤 5: 观察告警

**等待 10-15 秒后**，你应该看到:

1. **前端页面**:
   - "活动告警"区域出现新的告警卡片
   - 显示红色图标和告警信息
   - 包含节点名称、当前值、阈值等信息

2. **浏览器控制台**:
   ```
   Alert message received: {type: 'alert', action: 'triggered', data: {...}}
   ```

3. **后端日志**:
   ```
   Alert triggered: CPU使用率测试 on node001 (65.5% > 50%)
   Alert notification sent: event_xxx
   ```

### 步骤 6: 处理告警

1. 点击告警卡片上的"处理"按钮
2. 输入处理备注（可选）: "已确认，正在处理"
3. 点击确定

**预期结果**:
- 告警从"活动告警"列表中消失
- 后端日志显示: `Alert event resolved: event_xxx`

### 步骤 7: 测试自动恢复

1. 创建新规则（阈值设为 80）
2. 触发告警（让 CPU 超过 80%）
3. 等待告警出现
4. 停止 CPU 密集型任务
5. 等待 10-15 秒

**预期结果**:
- 告警自动从列表中消失
- 浏览器控制台显示: `Alert recovered: ...`
- 后端日志显示: `Recovery notification sent: ...`

## 🐛 常见问题

### 问题 1: 告警不触发

**可能原因**:
- 规则未启用
- 持续时间未满足
- 阈值设置不合理
- Agent 未上报数据

**解决方法**:
1. 检查规则是否启用
2. 降低持续时间（如 10 秒）
3. 降低阈值
4. 检查 Agent 是否正常运行

### 问题 2: 前端不显示告警

**可能原因**:
- WebSocket 未连接
- 前端未刷新数据
- 浏览器控制台有错误

**解决方法**:
1. 检查 WebSocket 连接状态
2. 点击"刷新"按钮
3. 查看浏览器控制台错误
4. 重新登录

### 问题 3: 告警不自动恢复

**可能原因**:
- 指标仍超过阈值
- 检测器未运行
- WebSocket 推送失败

**解决方法**:
1. 确认指标已恢复正常
2. 检查后端日志
3. 手动刷新页面
4. 重启后端服务

### 问题 4: 后端启动失败

**可能原因**:
- Redis 未启动
- InfluxDB 未启动
- 端口被占用

**解决方法**:
1. 启动 Redis
2. 启动 InfluxDB
3. 检查端口 50001 是否被占用
4. 查看后端错误日志

```bash
curl -X POST http://localhost:50001/api/alert/rules \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "name": "CPU使用率测试",
    "nodeId": "*",

    "duration": 10,
```
