## 🎯 核心功能

### 1. 系统健康检查 ✅

**功能描述**:
- 分析所有节点当前状态
- 评估系统整体健康度（0-100分）
- 识别关键问题和风险
- 提供即时优化建议

**输入数据**:
- 节点在线状态
- CPU/内存/磁盘使用率
- 活动告警数量
- 系统配置信息

**输出结果**:
- 健康评分
- 状态描述（优秀/良好/警告/危险）
- 问题列表（严重程度、节点、指标、描述）
- 优化建议列表
- 紧急程度评估

### 2. 性能趋势分析 ✅

**功能描述**:
- 分析历史性能数据
- 识别趋势方向和周期模式
- 预测未来资源使用
- 发现异常波动

**输入数据**:
- 节点ID
- 时间范围（24h/7d/30d）
- 历史指标数据

**输出结果**:
- 趋势方向（上升/下降/稳定）
- 周期模式识别
- 未来预测（30天）
- 关键洞察
- 异常检测

### 3. 优化建议 ✅

**功能描述**:
- 分析系统配置和资源使用
- 提供分类优化方案
- 评估影响和工作量
- 区分短期和长期建议

**输入数据**:
- 所有节点状态
- 资源使用情况
- 告警规则配置

**输出结果**:
- 快速见效建议
- 分类优化方案（资源配置、性能调优等）
- 长期规划建议
- 优先级和工作量评估

```
┌─────────────────────────────────────────────┐
│              前端 (Vue 3)                   │
│  ┌─────────────────────────────────────┐   │
│  │  AIAnalysis.vue (AI分析页面)        │   │
│  │  ├─ 健康检查                        │   │
│  │  ├─ 趋势分析                        │   │
│  │  └─ 优化建议                        │   │
│  └─────────────────────────────────────┘   │
│  ┌─────────────────────────────────────┐   │
│  │  ai.ts (API封装)                    │   │
│  └─────────────────────────────────────┘   │
│  ┌─────────────────────────────────────┐   │
│  │  ai.ts (Pinia Store)                │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
                    ↓ HTTP
┌─────────────────────────────────────────────┐
│            后端 (Node.js)                   │
│  ┌─────────────────────────────────────┐   │
│  │  ai.js (API路由)                    │   │
│  │  ├─ GET  /ai/status                 │   │
│  │  ├─ POST /ai/analyze/health         │   │
│  │  ├─ POST /ai/analyze/trend          │   │
│  │  └─ POST /ai/analyze/recommendations│   │
│  └─────────────────────────────────────┘   │
│  ┌─────────────────────────────────────┐   │
│  │  aiService.js (AI核心服务)          │   │
│  │  ├─ 数据收集                        │   │
│  │  ├─ 提示词构建                      │   │
│  │  ├─ LLM调用                         │   │
│  │  └─ 结果解析                        │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
        ↓ Redis          ↓ InfluxDB
┌──────────────┐   ┌──────────────┐
│  实时数据    │   │  历史数据    │
└──────────────┘   └──────────────┘
                    ↓ HTTP
            ┌──────────────┐
            │  LLM服务     │
            │  (Ollama)    │
            └──────────────┘
```

### 数据流

```
1. 用户点击分析按钮
   ↓
2. 前端调用API
   ↓
3. 后端收集数据
   ├─ Redis (实时状态)
   ├─ InfluxDB (历史数据)
   └─ alertService (告警信息)
   ↓
4. 构建分析提示词
   ↓
5. 调用LLM API
   ├─ OpenAI (云端)
   └─ Ollama (本地)
   ↓
6. 解析AI响应
   ├─ JSON提取
   ├─ 数据验证
   └─ 错误处理
   ↓
7. 返回分析结果
   ↓
8. 前端更新UI
   ├─ 健康评分
   ├─ 问题列表
   ├─ 优化建议
   └─ 分析历史
```

## 🎯 使用指南

### 快速启动

```bash
# 1. 启动后端
cd backend
node index.js

# 2. 启动Ollama
ollama serve

# 3. 启动前端
cd frontend
npm run dev

# 4. 访问系统
# http://localhost:5173
# 登录后点击 "AI智能分析"
```

### 配置说明

**后端配置** (`backend/.env`):
```env
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama2:7b
```

**前端配置**:
- 无需额外配置
- 自动连接后端API

### 使用流程

1. **系统健康检查**
   - 点击 "系统健康检查" 卡片
   - 等待10-30秒
   - 查看健康评分和建议

2. **性能趋势分析**
   - 点击 "性能趋势分析" 卡片
   - 选择节点和时间范围
   - 点击 "开始分析"
   - 查看趋势和预测

3. **优化建议**
   - 点击 "优化建议" 卡片
   - 等待10-30秒
   - 查看分类建议
