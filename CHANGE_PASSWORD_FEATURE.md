# 修改密码功能实现

## 功能概述

在系统设置页面添加了完整的修改密码功能，包括密码强度检测、弱口令警告、密码可见性切换等特性。

## 功能特性

### 1. 用户信息展示
- 显示当前登录用户的基本信息
- 用户名、角色、用户ID
- 角色标签美化显示

### 2. 修改密码表单
- 当前密码输入
- 新密码输入（带强度检测）
- 确认新密码输入
- 实时表单验证

### 3. 密码强度检测
- 实时显示密码强度（弱/中/强）
- 可视化强度条（红/橙/绿）
- 弱口令警告提示
- 密码不一致提示

### 4. 密码可见性切换
- 每个密码输入框都有显示/隐藏按钮
- 眼睛图标切换
- 提升用户体验

### 5. 安全验证
- 前端验证：长度、一致性、强度
- 后端验证：旧密码正确性、新密码强度
- 密码加密存储（bcrypt）

## 使用流程

### 修改密码步骤

1. **进入设置页面**
   - 点击侧边栏"设置"菜单
   - 查看用户信息

2. **填写表单**
   - 输入当前密码
   - 输入新密码（查看强度提示）
   - 确认新密码

3. **提交修改**
   - 点击"修改密码"按钮
   - 等待处理（按钮显示"修改中..."）
   - 查看成功/失败消息

4. **修改成功**
   - 显示绿色成功消息
   - 表单自动重置
   - 3秒后消息自动消失

## 表单验证规则

### 前端验证

1. **必填验证**
   - 所有字段必填
   - 空值时禁用提交按钮

2. **长度验证**
   - 新密码至少6位
   - 不满足时禁用提交按钮

3. **一致性验证**
   - 新密码与确认密码必须一致
   - 不一致时显示红色提示

4. **强度检测**
   - 实时显示密码强度
   - 弱口令显示警告（不阻止提交）

### 后端验证

1. **身份验证**
   - 验证JWT token有效性
   - 确认用户存在

2. **旧密码验证**
   - 使用bcrypt比对
   - 错误时返回401

3. **新密码验证**
   - 最小长度6位
   - 可扩展更多规则

## 密码强度评分

### 评分规则（与登录页面一致）

- **长度**：8位+25分，12位+15分，16位+10分
- **字符类型**：小写+15分，大写+15分，数字+15分，特殊字符+20分
- **多样性**：3种类型+10分，4种类型+10分
- **弱口令惩罚**：匹配弱口令强制降到30分以下

### 强度等级

- **弱密码**（0-39分）：红色
- **中等强度**（40-69分）：橙色
- **强密码**（70-100分）：绿色

## 错误处理

### 常见错误及处理

1. **当前密码错误**
   ```
   错误码: INVALID_OLD_PASSWORD
   提示: "当前密码不正确"
   ```

2. **新密码过短**
   ```
   错误码: WEAK_PASSWORD
   提示: "新密码至少需要6个字符"
   ```

3. **未登录**
   ```
   错误码: NO_TOKEN
   提示: "需要登录"
   ```

4. **Token过期**
   ```
   错误码: INVALID_TOKEN
   提示: "登录已过期，请重新登录"
   ```

## 安全建议

### 密码策略建议

1. **最低要求**
   - 至少8个字符
   - 包含大小写字母
   - 包含数字
   - 包含特殊字符

2. **避免使用**
   - 与用户名相同
   - 常见弱口令
   - 个人信息
   - 键盘序列

3. **最佳实践**
   - 定期更换密码（建议3-6个月）
   - 不同系统使用不同密码
   - 使用密码管理器
   - 启用多因素认证

## API文档

**请求**:
```http
POST /api/auth/change-password
Authorization: Bearer <token>
Content-Type: application/json

{
  "oldPassword": "admin",
```

**成功响应**:
```json
{
  "success": true,
  "message": "Password changed successfully"
}
```

**错误响应**:
```json
{
  "success": false,
  "error": {
    "code": "INVALID_OLD_PASSWORD",
```
